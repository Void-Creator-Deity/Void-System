## 1. 项目概述

### 1.1 项目简介

**虚空系统 (Void System)** 是一个集成了AI智能助手、文档管理、任务系统和个性化RAG问答功能的全栈应用。系统采用模块化设计，前后端分离，提供了丰富的API接口和用户友好的前端界面。

### 1.2 核心目标

- 提供高效的文档管理和智能检索功能
- 实现基于用户文档的个性化问答系统
- 构建完整的任务管理和属性成长体系
- 集成AI智能助手，提供多种交互模式

### 1.3 项目状态

✅ **已实现**：基础架构、用户管理、文档管理、向量搜索、个性化QA、任务系统

🔄 **开发中**：AI功能优化、性能调优、安全增强

📋 **计划中**：微服务架构迁移、Redis缓存、API速率限制

## 2. 系统架构

### 2.1 整体架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  前端应用层     │     │  后端服务层     │     │  数据存储层     │
│  Vue 3 + Element Plus │ │  FastAPI + LangChain  │ │  SQLite + ChromaDB  │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │                       │
        └───────────────────────┼───────────────────────┘
                                 │
                     ┌───────────┴───────────┐
                     │  AI 服务层            │
                     │  Ollama + Qwen3-14B   │
                     └───────────────────────┘
```

### 2.2 后端架构

```
void-system-backend/
├── api/                # API模块（向量管理、文档处理、个性化QA等）
├── chroma_db/          # 向量数据库
├── middleware/         # 中间件（认证等）
├── services/           # 业务服务（AI服务、用户服务等）
├── tests/              # 测试代码
├── tools/              # 工具函数
├── config.py           # 配置文件
├── database.py         # 数据库操作
├── errors.py           # 错误处理
└── main.py             # 应用入口
```

### 2.3 前端架构

```
void-system-frontend/
├── src/
│   ├── api/           # API调用封装
│   ├── components/    # Vue组件
│   ├── pages/         # 页面组件
│   ├── router/        # 路由配置
│   ├── App.vue        # 根组件
│   └── main.js        # 应用入口
```

| 技术/框架       | 用途                  | 版本/说明                          |
|---------------|----------------------|----------------------------------|

## 5. 代码质量与设计

### 5.1 代码组织

- **模块化设计**：功能模块清晰分离，便于维护和扩展
- **清晰的目录结构**：按功能分类，代码层次分明
- **统一的API响应格式**：所有API返回标准化的JSON格式
- **完善的错误处理**：统一的异常处理机制

### 5.2 设计模式与最佳实践

- **依赖注入**：使用FastAPI的Depends机制进行依赖管理
- **异步编程**：充分利用Python异步特性，提高性能
- **中间件设计**：实现了请求日志、认证等中间件
- **RESTful API设计**：遵循RESTful设计原则
- **配置管理**：集中式配置，支持环境变量

### 5.3 安全考虑

- **密码哈希存储**：使用bcrypt进行密码加密
- **JWT令牌认证**：安全的用户认证机制
- **输入验证**：使用Pydantic进行严格的输入验证
- **权限控制**：基于角色的访问控制

### 5.4 潜在安全隐患

⚠️ **待优化安全点**：
- 缺少API速率限制
- 未实现令牌黑名单机制
- 敏感信息日志记录需进一步优化
- 向量搜索结果需进行权限验证

```bash
cd void-system-backend
python -m uvicorn main:app --reload
```

### 前端启动

```bash
cd void-system-frontend
npm install
npm run dev
```

- 后端API: http://localhost:8000
- API文档: http://localhost:8000/api/docs
