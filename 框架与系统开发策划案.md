## 一、项目概述

### 1. 项目背景

现实生活中，人们常面临"目标感缺失、成长路径不清晰、缺乏持续动力"等问题。融合小说中金手指系统的"任务-属性-奖励"核心逻辑，结合现代AI技术能力，打造一款沉浸式个人成长辅助系统，通过"自定义任务→完成获得属性提升→系统币兑换资源"的闭环，让个人成长过程更具游戏感、成就感和高效性。

### 2. 核心定位

**"个人成长金手指系统"**：以小说中系统流的游戏化元素（属性、任务、系统币、资源兑换）为核心框架，以强大的AI辅助能力为技术支撑，营造"获得神秘系统辅助"的沉浸感，核心是让用户在个人成长道路上"目标明确、动力持久、成长可见"。

### 3. 目标用户

面向所有追求自我提升的人群，包括学生、职场人士、终身学习者等，支持自定义成长方向（如学习提升、技能培养、习惯养成、职业发展等），打造真正个性化的成长系统。

### 🚀 快速启动

#### 环境要求

- **Python**: >= 3.12
- **Node.js**: >= 16.0
- **Ollama**: 需要本地运行 Ollama 服务（用于 LLM 模型）

#### 后端启动

```bash
cd void-system-backend

# 使用 uv 管理依赖（推荐）
uv sync
uv run uvicorn main:app --reload --host 127.0.0.1 --port 8000

# 或使用传统方式
pip install -r requirements.txt
uvicorn main:app --reload
```

后端服务将在 `http://127.0.0.1:8000` 启动，API 文档访问：`http://127.0.0.1:8000/docs`

#### 前端启动

```bash
cd void-system-frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端服务将在 `http://localhost:5173` 启动

```bash
# 添加文件
git add .

git commit -m "提交说明"

git push -u origin main
```

## 四、技术方案

### 1. 技术栈选型（性能优先+体验优先）

| 模块         | 当前技术选择                      | 理由说明                                  |
|--------------|-----------------------------------|-------------------------------------------|
| 前端框架     | Vue 3 + Vite                      | 现代化、高性能、开发体验优秀               |
| UI组件库     | Element Plus                      | 组件丰富、文档友好，适合快速搭建界面       |
| HTTP客户端   | Axios                             | 功能完善、拦截器支持、错误处理             |
| 路由管理     | Vue Router                        | 官方路由方案、支持懒加载、路由守卫         |
| 后端框架     | FastAPI                           | 高性能、自动API文档、类型检查、异步支持    |
| 数据库       | SQLite（当前）                    | 无需部署、文件型数据库、适合初期开发       |
| ORM/数据库   | 原生SQLite（当前）                 | 轻量级、直接控制、适合MVP阶段              |
| AI框架       | LangChain + LangServe             | 专业的LLM应用框架、完整的RAG能力链         |
| 向量数据库   | ChromaDB                          | 轻量级、易于集成、支持本地部署             |
| LLM模型      | Ollama（本地）                    | 免费、可离线、支持多种模型                 |
| 认证方案     | JWT (jose)                        | 无状态、易于扩展、标准化                   |
| 密码加密     | bcrypt                            | 安全可靠、行业标准                         |
| 数据验证     | Pydantic                          | 类型安全、自动验证、FastAPI集成            |
| 部署方式     | 本地开发 → 云服务器部署           | 初期本地测试，后期支持在线访问             |

## 五、核心功能模块

### 1. 用户系统

- ✅ **基础功能**：注册/登录（用户名+密码，JWT认证）、密码加密存储（bcrypt）、用户信息管理
- ✅ **个人中心**：展示属性面板、任务进度、系统币余额、用户等级
- **核心价值**：建立用户专属数据空间，记录学习轨迹

### 2. 属性系统（核心特色）

- ✅ **自定义属性**：用户可创建个性化属性（如"高数熟练度""Python编程""雅思词汇量"），设置最大值（默认100）
- ✅ **属性成长**：完成关联任务后，通过"手动自评"或"AI建议"获得属性点，属性值实时展示（进度条/数值）
- ✅ **属性管理**：属性创建、查询、更新、删除
- **未来扩展**：属性等级系统、属性关联关系、属性技能树

### 3. 任务系统

- ✅ **任务创建**：用户自定义任务名称、目标描述、关联属性、预计耗时、奖励（系统币数量）
- ✅ **任务流程**：创建→标记"进行中"→完成后提交（可上传证明数据）→获得属性点+系统币
- ✅ **任务状态**：pending（待开始）→ in_progress（进行中）→ completed（已完成）/ failed（失败）
- ✅ **任务评估**：用户自评（完成度、质量评分），系统推荐属性点
- **未来扩展**：任务模板、任务链、任务依赖、AI自动评估、任务证明文件上传

### 4. 经济系统（激励闭环）

- ✅ **系统币获取**：完成任务固定奖励（可自定义奖励数量）
- ✅ **系统币记录**：完整的收支记录（earn/spend/reward类型）
- ✅ **商店系统**：物品兑换（AI服务次数、学习资料等）
- ✅ **资源管理**：用户资源表（ai_qa、ai_advisor、material等类型）
- **未来扩展**：每日签到奖励、成就系统奖励、任务链奖励、市场交易系统

### 5. AI赋能系统（核心金手指）

- ✅ **系统精灵（Persona Chain）**：多轮对话、会话历史管理、个性化建议
- ✅ **知识库问答（QA Chain）**：基于ChromaDB的RAG检索、文档嵌入、精准回答
- ✅ **任务建议（Advisor Chain）**：结构化任务建议生成、基于用户属性推荐
- **未来扩展**：
  - 多模态AI（图像识别、语音输入）
  - AI Agent能力（自动任务规划、执行）
  - 个性化模型微调
  - 多模型支持（切换不同LLM）

## 六、目标架构设计

### 1. 架构演进路线

#### 阶段一：MVP阶段（当前）✅

```text
单机部署
├── 前端：Vite开发服务器
├── 后端：FastAPI单进程
├── 数据库：SQLite（文件型）
└── AI服务：本地Ollama
```

#### 阶段二：扩展阶段（未来）

```text
分布式架构
├── 前端：CDN + 静态资源服务器
├── 后端：FastAPI多实例（负载均衡）
├── 数据库：PostgreSQL（主）+ Redis（缓存）
├── 向量数据库：ChromaDB集群 或 Milvus
├── AI服务：模型服务化（独立服务）
└── 消息队列：RabbitMQ / Celery（异步任务）
```

#### 阶段三：微服务架构（长期）

```text
微服务架构
├── API网关（Kong / Traefik）
├── 用户服务（独立服务）
├── 任务服务（独立服务）
├── 属性服务（独立服务）
├── 经济服务（独立服务）
├── AI服务集群
│   ├── LLM服务
│   ├── RAG服务
│   └── Agent服务
└── 数据服务
    ├── 关系数据库（PostgreSQL）
    ├── 向量数据库（Milvus）
    └── 缓存（Redis）
```

### 2. 数据库架构演进

#### 当前：SQLite（单文件）

- ✅ 适合MVP和开发阶段
- ✅ 无需额外部署
- ⚠️ 并发性能有限
- ⚠️ 不适合大规模数据

#### 未来：PostgreSQL + Redis

- **PostgreSQL**：主数据库，支持复杂查询、事务、并发
- **Redis**：缓存层，会话管理、热点数据、实时统计
- **数据迁移策略**：SQLite → PostgreSQL 迁移脚本

### 3. AI服务架构演进

#### 当前：本地Ollama

- ✅ 免费、可离线
- ✅ 适合开发测试
- ⚠️ 性能受限于本地硬件

#### 未来：AI服务化

- **模型服务层**：统一的LLM接口，支持多模型切换
- **RAG服务层**：独立的向量检索服务
- **Agent服务层**：任务规划、执行Agent
- **模型管理**：模型版本管理、A/B测试

## 十、风险与应对

| 风险点               | 影响程度 | 应对措施                                  |
|----------------------|----------|-------------------------------------------|
| 技术难点（如RAG效果差） | 中       | 初期用关键词检索保底，持续优化RAG策略，引入重排序机制 |
| 数据量不足（属性评估不准） | 低       | 依赖用户自评，积累数据后再训练简单规则模型，引入AI辅助评估 |
| 开发进度滞后          | 高       | 优先保证核心闭环（任务→属性→币），砍掉非必要功能，采用敏捷开发 |
| 数据库性能瓶颈        | 中       | 提前规划数据库迁移，添加缓存层，优化查询性能 |
| AI服务不稳定          | 中       | 多模型支持，降级策略，本地模型备份 |
| 用户增长带来的扩展压力 | 高       | 预留扩展接口，采用可扩展架构，提前规划微服务拆分 |
